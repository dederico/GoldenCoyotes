1. ✅ FILTRO DE OPORTUNIDADES POR RED DE CONTACTOS - COMPLETADO

   Problema: Cualquier usuario podía ver TODAS las oportunidades de la plataforma
   Solución: Ahora solo ves oportunidades de tu red de contactos

   Cambios críticos:
   - Modificado database_setup.py::get_opportunities()
   - Nuevo parámetro: network_only=True, requesting_user_id
   - Query SQL con INNER JOIN a tabla connections
   - Filtra por status = 'accepted'
   - Excluye oportunidades propias

   Query SQL implementado (database_setup.py:375-387):
   ```sql
   SELECT DISTINCT o.*, u.name as creator_name
   FROM opportunities o
   JOIN users u ON o.user_id = u.id
   INNER JOIN connections c ON (
       (c.user_id = ? AND c.connected_user_id = o.user_id)
       OR (c.connected_user_id = ? AND c.user_id = o.user_id)
   )
   WHERE c.status = 'accepted' AND o.is_active = 1 AND o.user_id != ?
   ORDER BY o.created_at DESC
   ```

   Vistas actualizadas:
   - get_public_opportunities() usa network_only=True
   - get_directed_opportunities() filtra por tags
   - opportunities_status combina red + mis oportunidades
   - get_ai_recommended_opportunities() solo de mi red

2. ✅ ENDPOINTS REST PARA GESTIÓN DE CONEXIONES - COMPLETADO

   Problema: No había endpoints dedicados para gestionar conexiones
   Solución: 5 endpoints REST transparentes

   Endpoints implementados:

   GET /api/connections?status=accepted
   - Obtener mis conexiones filtradas por estado
   - Retorna lista de conexiones con info de usuarios

   GET /api/connections/pending
   - Ver solicitudes de conexión pendientes (que me enviaron)
   - Retorna lista de solicitudes con datos del solicitante

   POST /api/connections/request
   Body: { "target_user_id": "uuid", "message": "opcional" }
   - Solicitar conexión con otro usuario
   - Validaciones: usuario existe, no eres tú mismo, no existe ya

   POST /api/connections/accept
   Body: { "connection_id": "uuid" }
   - Aceptar solicitud de conexión
   - Seguridad: solo puedes aceptar tus propias solicitudes

   POST /api/connections/reject
   Body: { "connection_id": "uuid" }
   - Rechazar solicitud de conexión
   - Seguridad: solo puedes rechazar tus propias solicitudes

   Archivo: golden_coyotes_after_feedback.py (líneas 397-513)

   get_pending_connection_requests(user_id)
   - Obtiene solicitudes pendientes RECIBIDAS por el usuario

   reject_connection(connection_id, user_id)
   - Rechaza una solicitud de conexión
   - Valida permisos antes de rechazar

   is_connected(user_id, target_user_id)
   - Verifica si dos usuarios están conectados (bidireccional)
   - Chequea status='accepted'
